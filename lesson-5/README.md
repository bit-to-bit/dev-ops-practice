# DevOps Practice — Lesson 5

Коротка документація проєкту.

## Структура проєкту
- modules/
  - s3-backend/ — конфігурація бекенду для стану Terraform (S3 bucket + DynamoDB для блокувань).
  - vpc/ — створення VPC: підмережі (public/private), route tables, Internet Gateway, NAT Gateway, security groups.
  - ecr/ — створення ECR репозиторіїв для зберігання образів Docker (політики життєвого циклу, шифрування, доступи).
- main.tf, variables.tf, outputs.tf — кореневі конфігураційні файли Terraform.
- backend.tf або відповідна конфігурація у s3-backend — налаштування збереження стану.

## Команди
- terraform init
  - Ініціалізація робочого каталогу, завантаження провайдерів і налаштування бекенду.
  - Якщо змінювалися налаштування бекенду: terraform init -reconfigure
- terraform plan
  - Перегляд плану змін (рекомендується із --var-file або відповідними змінними середовища).
- terraform apply
  - Застосування плану (можна додавати --auto-approve для автоматичного підтвердження).
- terraform destroy
  - Видалення створених ресурсів (--auto-approve можливо для автоматизації).

Примітка: виконувати команди в корені проєкту або в конкретному каталозі середовища; переконайтеся, що налаштовані AWS-креденшіали (AWS_PROFILE або AWS_ACCESS_KEY_ID / AWS_SECRET_ACCESS_KEY).

## Пояснення модулів
- s3-backend
  - Відповідає за збереження стану Terraform у S3 бакеті і використання DynamoDB таблиці для блокувань. Забезпечує спільний стан для багатьох користувачів/CI.
- vpc
  - Створює мережеву інфраструктуру: VPC, підмережі (public/private), маршрути, Internet Gateway та NAT Gateway; налаштовує базові security groups.
- ecr
  - Створює AWS ECR репозиторій(ї) для зберігання контейнерних образів, налаштовує політики доступу та життєвого циклу.
